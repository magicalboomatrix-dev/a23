## WALLET LOGIC

Wallet has:

1. balance (real money)
2. bonus_balance
3. exposure (calculated, not stored)

Exposure = sum of all active bets

Available Withdrawal = balance - exposure

RULES:

* Bet deducts from balance immediately
* Exposure increases
* On result:

  * Win → add winnings to balance
  * Loss → nothing added
* Bonus cannot be withdrawn

---

## GAME LOGIC

1. JODI:

* Numbers: 00–99
* User can select multiple numbers
* Each number has its own amount

2. HARUF:

* Numbers: 0–9
* Two modes:

  * Andar
  * Bahar

3. CROSSING:

* User inputs 1–7 digits
* Generate combinations:

WITH JODI:

* n × n combinations

WITHOUT JODI:

* n × (n - 1) combinations

Exclude same digits when without jodi:

* 11, 22, etc.

---

## BETTING RULES

* Admin sets payout ratios:

  * jodi
  * haruf
  * crossing

* Admin sets time-based max bet:
  Example:

  * 1 hour before → max ₹100
  * 30 min before → max ₹50

System must validate before placing bet.

## DEPOSIT SYSTEM

* User submits:

  * amount
  * UTR number
  * screenshot

RULES:

* UTR must be unique
* Duplicate UTR:

  * reject
  * notify admin & moderator

Moderator/Admin:

* approve or reject deposit

On approval:

* add amount to wallet

---

## WITHDRAW SYSTEM

* User submits withdraw request with bank account

VALIDATIONS:

* amount <= available withdrawal
* bonus not included

Approval:

* moderator or admin

On approval:

* deduct from wallet

Fraud Rule:

* same bank account used by multiple users → flag/block
